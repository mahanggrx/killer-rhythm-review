# 杀手节奏复盘反馈系统

用于游戏系统策划作品集的纯前端网页原型。项目读取人工制作的模拟对局日志，计算接敌、追击、发电机控制和永久减员结果指标，并通过确定性规则给出一条赛后练习建议。

当前版本已经完成从“选择、上传或按指标生成日志”到“获得唯一主要反馈”的最小可验证纵切，并提供适合作品集演示的响应式单页界面。

## 在线演示

[打开杀手节奏复盘反馈系统](https://mahanggrx.github.io/killer-rhythm-review/)

## 当前已实现

- 定义 `base_only_1v4_10.0.2` 基线下的 `MatchLog` 与 `MatchEvent` 判别联合类型。
- 使用 `eventId`、`timestampMs` 和 `eventOrder` 表达可追溯的严格事件顺序。
- 使用游戏正式 `chase_start/chase_end` 作为接敌与追逐依据，不再依赖人工 `target_acquired`；同时建模并发 `chaseId`、发电机回退/暂停/封锁以及玩家转 BOT 等机制。
- `parseMatchLogJson` 捕获损坏 JSON；`validateMatchLog` 返回结构错误和可见警告，不向页面抛出输入异常。
- `normalizeMatchLog` 按时间戳、事件顺序和原始位置稳定排序，且不修改原始输入。
- `calculateMatchMetrics` 通过统一入口输出四类指标；每项指标都有可用状态、解释、单位和证据事件 ID。
- 高进度阈值通过配置传入；重复事件有可追溯诊断，未闭合或删失追逐不会进入完整追逐均值。
- 详细指标口径见 `docs/metric-definitions.md`。
- `evaluateBreakpointRules` 根据 `src/config/rules.json` 评估“首次进入追逐较晚、平均追击时间较长、首次减员形成较晚”三类候选规则；无明显断点或证据不足时返回 `no_clear_breakpoint`。
- 规则建议文案与判断逻辑、阈值配置分离，详细口径见 `docs/rule-definitions.md`。
- 单页界面支持预置样例选择、点击或拖放上传 JSON 文件、按三个核心指标及可选高级参数生成合成日志、错误与警告展示、主要反馈、最多三个关键指标、可交互事件时间线和默认折叠的完整分析明细。
- 内置生成器支持自定义完整追逐次数、平均追逐时长、目标丢失或转火次数、高进度发电机达到阈值后未被控机即修开的数量、对达到阈值的高进度发电机的控机次数，以及最终永久减员数。这里的控机指杀手行为实际造成即时掉进度、开始回退或封锁。每项高级输入都显示具体数据口径；留空时使用确定性默认场景。结果必须通过现有日志校验与全部输入指标回算后才会进入分析，相同输入始终产生相同 JSON。
- 规则实验台支持开关三条规则、修改规则阈值和高进度发电机口径，并实时通过统一分析入口重新计算。第一版使用一套固定分析标准，不提供缺少数据校准依据的新手/进阶切换。
- 已提供“首次进入追逐较晚”“平均追击时间较长”“首次减员形成较晚”三份匿名合成样例，每份样例均有测试核对人工预期指标与主断点。
- 浏览器端不保存或上传日志；刷新页面后恢复默认样例和默认阈值。

## 环境要求

- Node.js 20.19+、22.12+ 或更高兼容版本
- npm

## 安装依赖

```bash
npm install
```

## 本地开发

```bash
npm run dev
```

命令启动后，打开终端显示的本地地址（通常是 `http://localhost:5173/`）。页面默认载入“首次进入追逐较晚”样例，可直接切换样例、上传本地 JSON，或点击“按指标生成日志”填写首次进入追逐时间、完整追逐平均时长和首次减员时剩余发电机。需要构造更复杂的对局时可展开“高级设置”；留空的高级参数由生成器自动补全。生成后会立即使用当前阈值重新分析。

## 运行测试

```bash
npm run test
```

持续监听测试：

```bash
npm run test:watch
```

## 类型检查

```bash
npm run typecheck
```

## 生产构建

```bash
npm run build
```

构建产物位于 `dist/`，可以部署到静态网站托管服务。

GitHub Pages 使用专用仓库路径构建：

```bash
npm run build:pages
```

推送到 `main` 后，`.github/workflows/deploy-pages.yml` 会自动执行测试、类型检查、构建与部署。

## 项目边界

- 只使用人工制作的模拟日志。
- 内置生成器产出的是规则测试用合成日志，不代表真实客户端事件记录或完整对局还原。
- 不接入或修改真实游戏客户端。
- 不使用后端、数据库、登录系统或机器学习。
- 默认规则集为 `base_only_1v4_10.0.2`。
- 所有阈值均属于原型待验证数值，不代表官方标准或真实平衡结论。
