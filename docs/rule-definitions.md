# 节奏断点规则口径（第一版）

适用规则集：`base_only_1v4_10.0.2`。

规则引擎接收已经计算完成的 `MatchMetrics`，输出唯一 `primaryFeedback` 和按确定性顺序排列的 `triggeredCandidates`。页面默认只应展示主反馈；候选列表用于解释与调试，不是多个同时展示的批评项。

所有默认阈值均来自 `src/config/rules.json`，属于“原型待验证数值”，不代表官方标准、真实平衡结论或经过真实玩家数据验证的判断线。

## 默认候选规则

| 规则 | 维度 | 默认触发条件 |
| --- | --- | --- |
| `FIRST_CHASE_TOO_LONG` | 追击 | `firstChaseToFirstHook > 75000ms`，且达到最低样本要求。该指标从首次正式追逐开始，只关联同一首追目标，包含追击、搬运和挂钩，不包含开局找人。 |
| `SEARCH_GAP_TOO_LONG` | 找人 | `averageSearchGap > 30000ms`，且达到最低样本要求。 |
| `GENERATOR_CONTROL_WEAK` | 发电机控制 | `highProgressGeneratorLosses >= 2`，且达到最低样本要求。 |
| `HOOK_PRESSURE_DIFFUSE` | 挂钩收益 | 总挂数至少 4，首挂获救后的再次上钩人数小于 1，至少存在 1 个转化机会，并且首次普通挂钩链减员晚于 300000ms 或明确没有普通挂钩链减员。 |

`HOOK_PRESSURE_DIFFUSE` 不会把任何“不可用的减员指标”都当作没有减员。只有原因明确为 `no_hook_chain_elimination` 才满足“没有普通挂钩链减员”；若缺少 `trial_start` 等导致无法计算，则证据不足且不触发。处决、流血死亡和无法确认原因的献祭不会替代该条件。

## 选择唯一主断点

所有触发候选使用以下稳定顺序比较：

1. 证据充分性，由有效样本量相对最低样本要求确定。
2. 相对阈值偏离。
3. 规则基础置信度与事件证据置信度的组合结果。
4. 当前经验层级的规则优先级。
5. 稳定的规则 ID 字典序。

默认经验优先级：

- `novice`：首次挂钩、再搜寻、控机、挂钩收益。
- `intermediate`：控机、挂钩收益、首次挂钩、再搜寻。

经验优先级只在证据充分性、相对阈值偏离和置信度相同时决定结果，界面会明确标注为“同等证据下优先”。

## 证据与输出

每条触发反馈包含：

- `ruleId`、标题、维度和中性严重度标签；
- 一条限定语气的说明和一个下一局可练习目标；
- 最多三个直接相关指标；
- 每项指标的状态、数值、单位、比较运算与阈值；
- 合并后的 `evidenceEventIds`，供页面定位时间线证据。

严重度标签只有 `moderate`、`high` 和 `critical`，不得转换为 S/A/B 评级或总分。

## 无明确断点

以下情况返回 `no_clear_breakpoint`：

- 没有规则超过配置阈值；
- 规则所需指标不可用或未达到最低样本；
- 规则配置无效。

无明确断点不是“表现优秀”的评级，只表示当前日志没有提供足够稳定的主要归因。

## 实现边界与尚未覆盖

- 规则结果已接入 React 页面和阈值编辑界面；阈值变化会通过统一分析入口重新计算。
- 规则配置目前是项目内 JSON；后续 UI 修改时仍需先通过同一运行时配置校验。
- 规则阈值尚未通过真实玩家样本回测或校准。
- 当前只有 `novice` 和 `intermediate` 两种经验上下文。
